# Voice & Preferences: Pradyumn Tendulkar

## Writing & Communication Style

### Formatting Rules (STRICT)
- **NEVER use em dashes (—)**. Use commas, periods, or "to" instead.
- Avoid words like "sidebar" and "tangent" in conversational writing.
- Don't use overly formal or stiff language. Warm but professional.
- Keep things concise. No filler.

### General Tone
- Direct. Doesn't beat around the bush.
- Prefers concrete examples over abstract descriptions.
- Values natural-sounding language. If a phrase sounds like something people wouldn't actually say in conversation, it should be rewritten.
- Analytical but accessible, opinionated but fair.
- Comfortable discussing both technical depth AND business implications.
- Frames controversial takes as questions or observations, not absolute declarations. Invites discussion rather than triggering defensive reactions.
- Hedges when citing research outside core domain. Uses "From what I've seen..." or "Based on X's data..." rather than stating external studies as settled fact.
- When challenged, redirects to the actual thesis rather than defending the technology or product being discussed. Stays on own territory.

### Professional Voice
- Leads with specific, quantified achievements.
- Professional but not corporate-stiff. Warm, direct, human.
- Signs off as "Pradyumn Tendulkar" formally, "Pradyumn" in warmer contexts.

## Design Preferences

### Typography
- Clean, modern sans-serif fonts (Inter, system font stack). Monospace for code/technical content.
- Font smoothing (antialiased rendering) expected.
- Strong typographic hierarchy: extrabold headings with tight tracking, relaxed body text with comfortable line-height.
- Typography should serve the content without competing with it.

### Color & Theme
- Light mode with subtle color accents is the primary aesthetic. Dark mode as a toggle option is welcome.
- Gradient accents (blue-to-purple) for visual interest on headings and key elements.
- Muted, professional base palette: gray-900 for text, gray-50/100 for backgrounds, blue-600 as primary accent.
- Semantic color usage: green for success, amber for warning, red for error. Every color should serve a purpose.
- Status indicators should be prominent and clear, not subtle.

### Layout & Structure
- Centered, max-width content areas for readability (max-w-4xl or similar).
- Generous spacing and padding. Breathable layouts, not cramped.
- Subtle borders with transparency (border-gray-100/200). Small consistent rounded corners (rounded-xl).
- Minimal shadows, used intentionally (shadow-lg with colored shadow like shadow-blue-600/20).
- Clean but not sterile. Professional with personality.
- Clear visual hierarchy through font weight, size, and whitespace.
- Section-based page structure with clear separation.

### Animations & Interactions
- Smooth, purposeful entrance animations (framer-motion fade+slide, staggered delays 0.1-0.3s).
- Subtle background effects: gradient blurs, soft radial glows.
- Pulse animations for status indicators, smooth hover transitions (transition-colors).
- Respects prefers-reduced-motion. No gratuitous animation.
- Loading states should show visual progress, not just spinners.

### Component Patterns
- Pill badges for labels/tags (rounded-full, colored background + border).
- Icon + text button patterns, not icon-only (lucide-react icons).
- Cards with hover states and subtle elevation changes.
- Inline elements over modals where possible.
- Composable component libraries (Shadcn/ui, Radix UI) preferred over opinionated frameworks.
- Proper focus states and accessibility (ring outlines, ARIA labels, color contrast).

### Tech Stack Preferences
- Next.js (App Router), TypeScript, Tailwind CSS for styling.
- framer-motion for animations.
- CSS variables for theming.
- Vercel for deployment.
